# 决策树算法 - 通俗易懂的例子

## 🎯 什么是决策树？

决策树就像一个"智能问答系统"，通过问一系列问题来做出决定。

### 生活中的例子：判断是否去户外玩

```
天气好吗？
├─ 是 → 有空闲时间吗？
│         ├─ 是 → 去玩！✓
│         └─ 否 → 不去 ✗
└─ 否 → 不去 ✗
```

## 📊 完整示例演示

### 示例数据

假设我们要预测一个学生是否会通过考试，有以下特征：
- **特征0**：是否完成作业（0=否，1=是）
- **特征1**：是否认真听课（0=否，1=是）
- **标签**：是否通过考试（0=不通过，1=通过）

**训练数据**：
```
学生1: [1, 1] → 通过(1)    # 完成作业，认真听课 → 通过
学生2: [1, 0] → 不通过(0)  # 完成作业，不认真听课 → 不通过
学生3: [0, 1] → 不通过(0)  # 不完成作业，认真听课 → 不通过
学生4: [0, 0] → 不通过(0)  # 不完成作业，不认真听课 → 不通过
```

### 算法执行过程

#### 步骤1：计算信息熵

**原始数据的熵**：
- 通过: 1个（25%）
- 不通过: 3个（75%）
- 熵 = -0.25×log₂(0.25) - 0.75×log₂(0.75) ≈ 0.81

这个0.81表示数据比较混乱（不是全通过或全不通过）

#### 步骤2：计算每个特征的信息增益

**特征0（是否完成作业）的信息增益**：

按特征0分组：
- 特征0=0: [[0,1,0], [0,0,0]] → 全是不通过 → 熵=0
- 特征0=1: [[1,1,1], [1,0,0]] → 有1个通过，1个不通过 → 熵=1

条件熵 = (2/4)×0 + (2/4)×1 = 0.5

信息增益 = 0.81 - 0.5 = **0.31**

**特征1（是否认真听课）的信息增益**：

按特征1分组：
- 特征1=0: [[1,0,0], [0,0,0]] → 全是不通过 → 熵=0
- 特征1=1: [[1,1,1], [0,1,0]] → 有1个通过，1个不通过 → 熵=1

条件熵 = (2/4)×0 + (2/4)×1 = 0.5

信息增益 = 0.81 - 0.5 = **0.31**

两个特征信息增益相同，选择索引更小的**特征0**

#### 步骤3：构建决策树

```
特征0（是否完成作业）
├─ 0（不完成）→ 不通过(0)
└─ 1（完成）→ 特征1（是否认真听课）
              ├─ 0（不认真）→ 不通过(0)
              └─ 1（认真）→ 通过(1)
```

决策树的数据结构：`(0, 0, (1, 0, 1))`
- 0: 根节点使用特征0
- 0: 特征0=0时，预测为0（不通过）
- (1, 0, 1): 特征0=1时，继续判断特征1
  - 1: 使用特征1
  - 0: 特征1=0时，预测为0
  - 1: 特征1=1时，预测为1

#### 步骤4：预测新学生

**新学生A**: [1, 1]（完成作业，认真听课）
1. 看特征0: 值是1 → 走右边
2. 看特征1: 值是1 → 走右边
3. 到达叶子节点：**1（通过）**

**新学生B**: [0, 1]（不完成作业，认真听课）
1. 看特征0: 值是0 → 走左边
2. 到达叶子节点：**0（不通过）**

## 🔍 核心概念解释

### 1. 信息熵（Entropy）

**作用**：衡量数据的"混乱程度"

**形象理解**：
- 🎯 全是同一类 → 熵=0（很整齐，不混乱）
- 🎲 一半一半 → 熵=1（最混乱）
- 📊 3个0，1个1 → 熵=0.81（有点混乱）

**公式**：`Entropy = -p₀×log₂(p₀) - p₁×log₂(p₁)`

### 2. 信息增益（Information Gain）

**作用**：判断哪个特征最有用

**形象理解**：
- 信息增益 = 使用特征前的混乱度 - 使用特征后的混乱度
- 信息增益越大，说明这个特征越能帮助我们分类
- 我们总是选择信息增益最大的特征来分割数据

**公式**：`增益 = 原始熵 - 加权平均条件熵`

### 3. 递归构建

决策树的构建是递归的：
1. 选择最好的特征
2. 按这个特征分割数据
3. 对每个子集重复步骤1-2
4. 直到所有数据属于同一类，或没有特征可用

## 💻 程序输入输出格式

### 输入格式：
```
n m                    # n=训练样本数，m=特征数
x₁₁ x₁₂ ... x₁ₘ y₁    # 第1个训练样本的m个特征 + 标签
x₂₁ x₂₂ ... x₂ₘ y₂    # 第2个训练样本的m个特征 + 标签
...
xₙ₁ xₙ₂ ... xₙₘ yₙ    # 第n个训练样本的m个特征 + 标签
q                      # q=查询数量
x₁₁ x₁₂ ... x₁ₘ       # 第1个查询样本的m个特征
x₂₁ x₂₂ ... x₂ₘ       # 第2个查询样本的m个特征
...
xq₁ xq₂ ... xqₘ       # 第q个查询样本的m个特征
```

### 输出格式：
```
y₁    # 第1个查询样本的预测标签
y₂    # 第2个查询样本的预测标签
...
yq    # 第q个查询样本的预测标签
```

### 完整示例：

**输入**：
```
4 2
1 1 1
1 0 0
0 1 0
0 0 0
2
1 1
0 1
```

**解释**：
- 4个训练样本，每个有2个特征
- 训练数据：[1,1]→1, [1,0]→0, [0,1]→0, [0,0]→0
- 2个查询：[1,1] 和 [0,1]

**输出**：
```
1
0
```

**解释**：
- [1,1] → 预测为1（通过）
- [0,1] → 预测为0（不通过）

## 🎓 算法优缺点

### ✅ 优点：
1. **易于理解**：决策过程直观，像人类思考一样
2. **无需归一化**：不需要对数据进行复杂的预处理
3. **可处理离散数据**：特别适合是/否这种二元特征

### ❌ 缺点：
1. **容易过拟合**：对训练数据拟合太好，新数据可能预测不准
2. **不稳定**：数据稍有变化，树的结构可能完全不同
3. **贪心算法**：每次选择局部最优，不一定全局最优
